## 1

1. חיבור בין בריפו הגלובלי למקומי
2. origin נוח לעבודה בצוותים
3. כי בתרגיל מקטן לא תמיד צריך גיבוי בענן

## 2

1. git remote add origin url
2. git push
3. הוא לא מכיר לאיפה לשלוח

## 3

1. git clone url
2. git pull
3. הראשון להתחבר בכלל השני לקבל שינויים שנעשו

## 4

1. עם https או ssh-keygen -t ed25519
2. שהחיבור נעשה אוטומטית
3. מחרוזת תווים בהגדרות החשבון משתמשים ב https

## 5

1. git clone url
2. שאני באמת מחובר למקום הנכון
3. git pull
4. git push
5. git init / git remote add origin url

## שאלות סיכום

1. git remote = לחבר פרויקט מהמחש שלי לענן
   git push = לשמור שינויים בריפו שכבר מחובר
   git clone = לחבר פרוייקט מהענן למחשב
2. זה ההענף המקביל לענף שאני עוד עליו במחשב שלי זה חשוב כי ככה בכל פקודה של העלאה או משיכה לא צריך לכתוב את שם הענף
3. במקרים שבהמשך נעבוד בו עם צוות או שתריך גיבוי
